From LLMs to Agents

Block 1 — 10 min

What Makes Something "Agentic"?

It's a spectrum, not a binary.

Five key properties

  1. Goal-directed behavior
  2. Tool use
  3. Planning & reasoning
  4. Memory
  5. Environment interaction
ChatGPT answering a question = not agentic
AI that researches, drafts, revises, and submits a PR = agentic
Start by asking "Is ChatGPT an agent?" — use responses to motivate the spectrum.

The ReAct Loop: Reason + Act

Observe
→
Think
→
Act
→
Observe
↺
  • Simple chain: fixed sequence — input → LLM → output
  • ReAct: dynamic loop — the model decides what to do next
  • The loop is non-deterministic — same input can produce different paths
  • This is THE fundamental pattern behind every agent framework
Stress the non-determinism. This is what makes agents fundamentally different from pipelines.

Tool Use / Function Calling

The LLM never "runs" code — it requests a function call, the runtime executes it.

User
"Do I need an umbrella in Amsterdam today?"
LLM emits
{"function": "get_weather", "args": {"city": "Amsterdam"}}
Runtime returns
{"temp": 14, "rain_prob": 0.85}
LLM responds
"Yes — 85% chance of rain. Bring an umbrella."
Tool definitions are part of the system prompt. The model "learns" what tools are available at inference time.

Why Not Just Write a Traditional Program?

Traditional Program

  • Every path must be pre-programmed
  • Combinatorial explosion
  • Brittle when inputs are ambiguous

Agent

  • Handles ambiguity with judgment
  • Adapts based on intermediate results
  • Composes tools creatively
Rule of thumb: clear, fixed procedure → write a program. Requires judgment & adaptation → consider an agent.

Trade-off: agents are less predictable, harder to debug, more expensive to run.

"Sort a list" = write a program. "Investigate why CI broke" = agent territory. Be honest about trade-offs.

Multi-Agent Patterns

Block 2 — 10 min

Why Multi-Agent?

Single agent limits

  • Context window bloat
  • Tool overload (50+ tools = confusion)
  • Conflicting instructions

Multi-agent wins

  • Specialization — focused roles
  • Modularity — develop independently
  • Parallel execution
Think: software team vs. one developer doing everything.
Relate to separation of concerns, single responsibility principle.

Triage Agent receives message → classifies intent (billing / technical / returns) → hands off to specialist with full history

Key question: who decides when to hand off?
The agent itself vs. explicit rules

Simplest multi-agent pattern. The handoff is itself a tool call.

LangGraph + Workshop

Block 3 — 10 min

LangGraph Core Concepts

  • Graph — the overall workflow definition
  • Nodes — individual steps (LLM call, tool execution, logic)
  • Edges — connections between nodes (fixed or conditional)
  • State — shared typed dictionary flowing through the graph
Why graphs? Explicit control flow — easier to debug and visualize than free-form loops.
Relate to FSMs or dataflow graphs from coursework.

create_react_agent

The 5-line agent

from langgraph.prebuilt import create_react_agent

tools = [search_tool, calculator_tool]
agent = create_react_agent(model, tools)
result = agent.invoke({"messages": [("user", "...")]})
  • Under the hood: builds the Observe-Think-Act graph automatically
  • Good starting point — customize when you need more control
  • This is what we start with in the workshop
Demystify: "an agent is just a graph with an LLM node and a tool node in a loop."
